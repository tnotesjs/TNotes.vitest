# 错误汇总

## 0002. Vitest 是什么

- [x] [3] 事实性错误：原文称 Vitest 是"基于 Vite 的单元测试框架"，但官方文档的定位是"由 Vite 驱动的下一代测试框架"。Vitest 不仅仅是单元测试框架，还支持组件测试、浏览器模式、基准测试、类型测试等，"单元测试框架"这一描述过于狭窄，应改为"测试框架"。
- [x] [3.1] 歧义/事实性错误：原文称"热模块替换：Watch 模式下修改代码立即重新测试"。官方文档描述为"智能文件监听模式，就像是测试的 HMR"，Watch 模式并非真正的 HMR（热模块替换），而是智能文件监听后重新运行受影响的测试。表述容易让读者误解 Vitest 支持真正的 HMR 机制，应修正为"智能监听：Watch 模式下智能监听文件变化，自动重新运行受影响的测试"。
- [x] [6.2] 事实性错误：表格中 Vitest 的 HMR 列写"支持热模块替换"，与 [3.1] 同样的问题，Watch 模式的智能监听不等于 HMR。

## 0006. 安装 Vitest

- [x] [7] 事实性错误：浏览器模式安装命令写的是 `npm install -D @vitest/browser playwright`，但官方文档推荐安装特定的 provider 包，如 `@vitest/browser-playwright` 或 `@vitest/browser-webdriverio`，而不是直接安装 `playwright` 包。正确写法应为 `npm install -D @vitest/browser @vitest/browser-playwright`，或使用官方推荐的 `npx vitest init browser` 命令自动安装。
- [x] [9] 歧义/事实性错误：原文称"TypeScript 支持、ESM 支持、路径别名解析等都由 Vitest 内置提供"。其中"路径别名解析"表述有歧义——Vite 的 `resolve.alias` 配置确实会被 Vitest 继承，但 `tsconfig.json` 中的 `paths` 路径别名需要额外安装 `vite-tsconfig-paths` 插件才能生效，并非完全内置。应明确区分两种路径别名。
