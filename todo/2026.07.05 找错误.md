# 找错误

- 查找范围：0002-0138
- 查找流程：
  - 从 0002 开始逐小节查找，一个小节就是一个二级标题
  - 将找到的错误汇总到 "todo/2026.07.05 错误汇总.md" 文件中
  - 记录的格式：
    - 笔记标题（每一篇笔记用一个二级标题来定义，这个二级标题下存放的是这一小节待修复的错误）
      - 错误的小节 + 错误的描述
      - 示例：`- [ ] [4] xxx`
      - 表示这篇笔记的第四小节有错误，错误描述为 xxx
  - 错误修复
- 注意事项：
  - 先查找错误（0002-0138）
  - 所有错误都查找完之后再执行修复
  - 修复完成之后将错误的复选框打勾
- 错误的判断标准：
  - 事实性错误：以 vitest 官方文档的描述为准，https://cn.vitest.dev/guide/，如果和官方文档描述的含义有明显冲突，则判定为错误
  - 阅读起来理解存在歧义的错误，修正表述，确保描述精确
  - 引号使用错误：在完整的中文语句中，不允许出现英文引号，比如 `"xxx"`，应该改为 `“xxx”`
