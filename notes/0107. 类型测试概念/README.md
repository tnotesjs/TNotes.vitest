## 2. 评价

类型测试确保类型定义的正确性，适合库项目维护公开 API 的类型稳定性。Vitest 内置了类型测试支持。

## 3. 什么是类型测试

类型测试验证 TypeScript 类型定义是否正确，而不是验证运行时行为。

```typescript
import { expectTypeOf } from 'vitest'

// 验证函数返回类型
function add(a: number, b: number): number {
  return a + b
}

expectTypeOf(add).returns.toBeNumber()
```

## 4. 与运行时测试的区别

| 维度     | 运行时测试 | 类型测试   |
| -------- | ---------- | ---------- |
| 测试内容 | 运行时行为 | 类型定义   |
| 执行方式 | 运行代码   | 编译检查   |
| 失败时机 | 运行时     | 编译时     |
| 测试对象 | 逻辑正确性 | 类型正确性 |

## 5. 适用场景

- 验证库的公开 API 类型
- 确保类型推断正确
- 捕获类型回归
- 文档化类型行为

## 6. 为什么需要类型测试

- 类型定义可能被意外修改
- 泛型约束可能被破坏
- 重载函数可能丢失类型
- 类型推断可能变弱

## 7. 测试内容

类型测试可以验证：

- 函数参数类型
- 函数返回类型
- 泛型约束
- 类型推断
- 重载解析

## 8. 工具选择

Vitest 内置了类型测试支持：

- `expectTypeOf`：类型断言
- `assertType`：类型检查

## 9. 与 dtslint 的对比

| 维度     | Vitest 类型测试 | dtslint  |
| -------- | --------------- | -------- |
| 配置     | 简单            | 复杂     |
| 集成     | 与 Vitest 集成  | 独立工具 |
| 学习成本 | 低              | 中       |
| 功能     | 基本满足        | 更丰富   |

## 10. 注意事项

- 类型测试不运行代码
- 类型测试在编译时检查
- 类型测试文件使用 `.test-d.ts` 后缀
