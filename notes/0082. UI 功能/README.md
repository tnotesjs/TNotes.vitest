## 2. 评价

UI 功能覆盖了测试开发的核心需求，实时更新配合 Watch 模式提高开发效率。

## 3. 测试列表

UI 左侧显示所有测试文件的树形结构：

- 按目录分组
- 显示测试状态（通过/失败/跳过）
- 显示执行时间

## 4. 失败详情

点击失败的测试可以查看详细信息：

- 错误消息
- 期望值 vs 实际值
- 堆栈跟踪
- 源代码定位

## 5. 筛选功能

UI 提供多种筛选方式：

- 按状态筛选（通过/失败/跳过）
- 按名称搜索
- 按文件筛选

## 6. 重跑测试

可以单独重跑某个测试或测试文件：

- 点击测试旁边的重跑按钮
- 右键菜单选择重跑
- 快捷键重跑

## 7. 测试详情

点击测试可以查看：

- 测试代码
- 执行时间
- 快照对比（如果有）
- 控制台输出

## 8. 实时更新

Watch 模式下，UI 会实时更新：

- 文件变化时自动重跑
- 测试结果实时刷新
- 错误信息实时更新

## 9. 快捷操作

UI 支持键盘快捷键：

- `r` - 重跑当前测试
- `f` - 只运行失败的测试
- `a` - 运行所有测试
- `s` - 搜索测试

## 10. 控制台输出

测试中的 `console.log` 输出会在 UI 中显示，方便调试。
