## 1. 本节内容

- 测试金字塔模型
- 单元测试（大量/快速）
- 集成测试（适量）
- 组件测试（适量）
- E2E 测试（少量/慢）
- 比例建议（70/20/10）
- 实际调整

## 2. 评价

- 测试金字塔提供了测试分配的指导原则
- 底层测试数量多、速度快、成本低；顶层测试数量少、速度慢、成本高
- 应根据项目特点灵活调整

## 3. 测试金字塔模型

```
        /  E2E  \          少量
       / 组件测试 \         适量
      / 集成测试   \        较多
     / 单元测试     \       大量
```

## 4. 单元测试

- 测试单个函数或模块
- 速度快，成本低
- 数量最多

## 5. 集成测试

- 测试多个模块的协作
- 速度适中
- 数量适中

## 6. 组件测试

- 测试 UI 组件的渲染和交互
- 速度适中
- 数量适中

## 7. E2E 测试

- 测试完整的用户流程
- 速度慢，成本高
- 数量最少

## 8. 比例建议

| 测试类型 | 比例 |
| -------- | ---- |
| 单元测试 | 70%  |
| 集成测试 | 20%  |
| E2E 测试 | 10%  |

## 9. 实际调整

根据项目特点调整比例：

- 纯函数库：几乎全部单元测试
- 前端组件库：大量组件测试
- 全栈应用：均衡分布

## 10. 注意事项

- 金字塔是指导原则，不是严格规则
- 根据项目需求调整比例
- 确保每层测试都有覆盖
