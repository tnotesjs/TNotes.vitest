## 2. 评价

更新快照使用 -u 参数，支持全量更新和按名称筛选更新。更新前应确认变化是预期的。

## 3. -u 参数

使用 `-u` 或 `--update` 更新快照：

```bash
# 更新所有快照
vitest run -u

# 更新指定文件的快照
vitest run src/utils.test.ts -u
```

## 4. 交互式更新

```bash
vitest run --update
```

交互式模式会逐个询问是否更新快照。

## 5. 更新外部快照

`.snap` 文件中的快照：

```bash
vitest run -u
```

会更新 `__snapshots__/` 目录下的 `.snap` 文件。

## 6. 更新内联快照

`toMatchInlineSnapshot` 的快照：

```bash
vitest run -u
```

会直接修改测试文件中的内联快照内容。

## 7. 只更新特定测试

```bash
# 只更新名称包含 "snapshot" 的测试
vitest run -t "snapshot" -u
```

## 8. 检查快照状态

```bash
# 不更新，只检查哪些快照过时
vitest run
```

过时的快照会导致测试失败。

## 9. 删除过时快照

```bash
vitest run --passWithNoTests
```

或者手动删除 `__snapshots__/` 目录中的无用文件。

## 10. 注意事项

- 更新快照前确保变化是预期的
- 更新后检查 diff
- 提交快照文件到版本控制
- CI 中不应自动更新快照
